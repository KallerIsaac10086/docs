## 目录结构

```
frontend/
├── src/
│   ├── lib/
│   │   ├── api/              # API 客户端
│   │   │   ├── client/       # 自动生成的 API 客户端
│   │   │   ├── files.ts      # 文件 API
│   │   │   └── openapi.json  # OpenAPI 规范
│   │   ├── components/       # UI 组件
│   │   │   ├── app/          # 应用组件
│   │   │   └── ui/           # 基础 UI 组件
│   │   ├── constants/        # 常量定义
│   │   ├── enums/            # 枚举类型
│   │   ├── hooks/            # 自定义 Hooks
│   │   ├── markdown/         # Markdown 处理
│   │   ├── services/         # 业务服务
│   │   ├── stores/           # 状态管理
│   │   ├── types/            # TypeScript 类型
│   │   └── utils/            # 工具函数
│   ├── routes/               # 页面路由
│   ├── app.css               # 全局样式
│   └── app.html              # HTML 模板
├── package.json
└── svelte.config.js
```

## 技术栈

| 技术 | 说明 |
|------|------|
| SvelteKit | 全栈前端框架 |
| Svelte 5 | 响应式组件库 |
| TailwindCSS | 原子化 CSS 框架 |
| TypeScript | 类型安全 |
| Dexie.js | IndexedDB 包装库 |
| Bits UI | 无头组件库 |
| Lucide | 图标库 |

## 核心模块

### 1. API 客户端 (lib/api/)

自动生成的 API 客户端，基于 OpenAPI 规范：

```bash
npm run gen:api
```

### 2. UI 组件 (lib/components/)

#### 应用组件 (app/)
- ChatForm - 聊天表单
- ChatMessage - 聊天消息
- ConversationList - 对话列表
- ModelSelector - 模型选择器

#### 基础组件 (ui/)
- Button - 按钮
- Card - 卡片
- Dialog - 对话框
- Input - 输入框
- Select - 选择器
- Switch - 开关
- Table - 表格
- 等等...

### 3. 状态管理 (lib/stores/)

使用 Svelte 5 的 `$state` 进行状态管理：

- auth.svelte.ts - 认证状态
- chat.svelte.ts - 聊天状态
- conversations.svelte.ts - 对话列表
- models.svelte.ts - 模型列表
- server.svelte.ts - 服务器状态
- settings.svelte.ts - 设置
- usage.svelte.ts - 用量统计
- cart.svelte.ts - 购物车
- private-chat.svelte.ts - 私有对话

### 4. 业务服务 (lib/services/)

- chat.ts - 聊天服务
- cloud.ts - 云端同步服务
- database.ts - 数据库服务
- models.ts - 模型服务
- tools.ts - 工具服务
- props.ts - 属性服务

### 5. 工具函数 (lib/utils/)

- api-headers.ts - API 请求头
- api-key-validation.ts - API Key 验证
- attachment-display.ts - 附件显示
- attachment-type.ts - 附件类型
- clipboard.ts - 剪贴板
- conversation-utils.ts - 对话工具
- file-preview.ts - 文件预览
- formatters.ts - 格式化
- model-names.ts - 模型名称
- pdf-processing.ts - PDF 处理
- 等等...

## 开发命令

```bash
# 开发模式
npm run dev

# 构建生产版本
npm run build

# 预览构建结果
npm run preview

# 类型检查
npm run check

# 代码格式化
npm run format

# 代码检查
npm run lint

# 运行测试
npm run test

# 生成 API 客户端
npm run gen:api

# Storybook
npm run storybook
```

## 环境变量

创建 `.env` 文件：

```env
# 后端 API 地址
VITE_API_URL=http://localhost:7878

# 其他配置
VITE_APP_NAME=LawChat
```

## 代理配置

前端通过 Vite 代理将 API 请求转发到后端服务：

```typescript
// vite.config.ts
export default defineConfig({
  server: {
    proxy: {
      '/v1': 'http://localhost:7878',
      '/admin-api': 'http://localhost:7878',
      '/api': 'http://localhost:7878',
      '/tools': 'http://localhost:7878',
      '/props': 'http://localhost:7878',
      '/users': 'http://localhost:7878',
      '/me': 'http://localhost:7878',
      '/token': 'http://localhost:7878',
      '/register': 'http://localhost:7878',
      '/cloud': 'http://localhost:7878',
      '/chats': 'http://localhost:7878',
      '/status': 'http://localhost:7878',
      '/checkout': 'http://localhost:7878',
      '/pay': 'http://localhost:7878',
      '/transactions': 'http://localhost:7878',
      '/purchase': 'http://localhost:7878',
      '/subscribe': 'http://localhost:7878',
      '/billing': 'http://localhost:7878',
      '/subscriptions': 'http://localhost:7878',
      '/plans': 'http://localhost:7878',
      '/vouchers': 'http://localhost:7878',
      '/shop': 'http://localhost:7878',
      '/upload': 'http://localhost:7878',
      '/usage': 'http://localhost:7878'
    }
  }
})
```

注意：`/admin` 是前端 SvelteKit 路由，不进行代理。
