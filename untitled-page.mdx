## 概述

**Law Query** 是 LawChat 内置的高精度法律案例检索引擎。它突破了传统关键词匹配的局限，通过"精确检索 + 语义理解 + 时序过滤"的三维架构，为您提供带有真实出处的法律依据。

在 LawChat 中，您无需手动切换检索模式。系统后台会自动分析您的提问意图，无感调用最优检索策略，确保每一次回答都**有理有据、拒绝幻觉**。

## 核心能力

Law Query 专为解决法律咨询中的"依据缺失"与"信息滞后"痛点而生：

- **权威案例溯源**
  目前已接入**最高人民法院、最高人民检察院**发布的全部指导案例。系统不仅提供案例名称，更直接展示**完整裁判文书**及**裁判要旨**，杜绝引用虚假案例。

- **智能语义匹配**
  不再依赖死板的关键词。即使您用口语化描述案情（如"房东不退押金怎么办"），系统也能理解核心法律关系，精准匹配相似案例。

- **动态时效过滤**
  支持按时间范围（如"近五年"、"2024 年以后"）筛选案例，确保您获取的信息**即时有效**，避免因法规变动导致决策失误。

- **一键核验详情**
  检索结果中的每一个引用均支持**一键跳转**。点击即可查看全文详情、裁判要旨及法院观点，让您可以像律师一样快速核验信息来源。

## 在 LawChat 中的使用

Law Query 已深度集成至 LawChat 的对话流程中，您只需自然语言交互，即可享受专业级检索服务。

### 启用工具调用

在 LawChat 输入框上方，点击 **+** 号，您将看到：

- **上传文档**：上传您的合同、案卷等材料
- **启用工具调用**：开启后，系统将自动调用 Law Query 进行案例检索
- **可用工具**：查看当前可调用的功能
  - **法律案例检索**：搜索相关指导案例
  - **获取案例详情**：查看具体案例的完整内容

> **提示**：建议始终保持"启用工具调用"处于开启状态，这样在您询问具体法律问题时，系统会自动检索相关案例作为依据。

### 自动意图识别

当您提出问题时，LawChat 的 **Auto 智能调度引擎** 会实时分析语义，自动选择最佳检索策略：

| 您的提问方式 | 系统自动调用的检索策略 | 返回结果特点 |
| :--- | :--- | :--- |
| **"帮我查一下'张三诉李四借款案'"** | **关键词精确检索** | 直接定位特定案件或当事人，返回精准判决书 |
| **"最近两年关于'AI 生成内容版权'的判例"** | **语义 + 时序组合检索** | 跨关键词匹配相似案情，并自动过滤掉两年前的旧案 |
| **"这种情况一般怎么判？"** | **深度语义检索** | 忽略具体措辞，分析核心法律关系，推送高相似度案例 |
| **"我想看这个案子的全文"** | **详情获取** | 直接调取并展示判决书全文、裁判要旨及争议焦点 |

### "核验溯源"交互设计

为了让结论可信可查，LawChat 将 Law Query 的检索结果渲染为**可交互组件**：

- **角标引用**：回答中的关键结论旁会标注来源角标（如 `[1]`、`[2]`）
- **悬浮预览**：鼠标悬停或点击角标，即刻弹出案例摘要卡片
- **全文跳转**：点击卡片中的"查看详情"，直接跳转至完整的裁判文书页面

> **数据来源**：所有检索案例均源自**最高人民法院指导案例库**、**最高人民检察院指导案例库**等权威渠道，确保数据的真实性与权威性。

## 常见问题

**Q: 检索到的案例如果是废止的怎么办？**  
A: Law Query 内置了**案例效力实时校验机制**。若某案例已被新司法解释或指导案例替代，系统会自动标注"已更新"，并优先推荐最新的相关案例，同时提示新旧案例的差异。

**Q: 我可以只检索特定地区的案例吗？**  
A: 可以。您在提问时加上地域限制（如"北京的离婚判例"），Law Query 会自动激活地域过滤器，仅返回该辖区法院的裁判文书。

**Q: 检索功能需要额外收费吗？**  
A: Law Query 的基础检索功能已包含在您的 **LawChat 会员权益** 中。高频次批量导出可能会消耗少量额外积分，具体请参阅 [计费逻辑](/docs/billing) 页面。

**Q: 为什么有时候检索不到我想要的案例？**  
A: 目前我们主要收录最高法、最高检的指导案例。如果您需要检索地方法院的具体案例，建议：1）使用更精确的案号或当事人名称；2）等待我们的数据库扩容（预计下季度上线地方案例库）。
