## iOS 部署

### 环境要求

- Xcode 15+
- macOS 14+
- iOS 13+ 目标设备

### 开发步骤

#### 1. 打开项目

```bash
cd flutter_app
open LawChat.xcodeproj
```

#### 2. 配置后端 API

编辑 `ContentView.swift` 中的 API 基础 URL：

```swift
let apiURL = "http://your-backend-url:7878"
```

#### 3. 选择目标设备

在 Xcode 顶部选择目标设备或模拟器。

#### 4. 运行项目

点击 Xcode 的 Run 按钮或按 `Cmd + R`。

### 发布到 App Store

#### 1. 配置签名

在 Xcode 中：
1. 选择项目 → Signing & Capabilities
2. 选择开发团队
3. 配置 Bundle Identifier

#### 2. 构建发布版本

Product → Archive

#### 3. 上传到 App Store Connect

Window → Organizer → Distribute App

#### 4. 填写应用信息

在 App Store Connect 中：
1. 应用名称和描述
2. 截图和图标
3. 隐私政策
4. 价格和可用地区

#### 5. 提交审核

提交应用供 Apple 审核。

## Android 部署

### 环境要求

- Android Studio
- JDK 11+
- Android 8.0+ 目标设备

### 开发步骤

#### 1. 打开项目

```bash
cd flutter_frontend
```

使用 Android Studio 打开项目。

#### 2. 配置后端 API

编辑 `MainActivity.kt` 中的 API 基础 URL：

```kotlin
const val API_URL = "http://your-backend-url:7878"
```

#### 3. 配置签名

在 `build.gradle` 中配置签名：

```gradle
android {
    signingConfigs {
        release {
            storeFile file("keystore.jks")
            storePassword "your-password"
            keyAlias "your-key-alias"
            keyPassword "your-key-password"
        }
    }
    
    buildTypes {
        release {
            signingConfig signingConfigs.release
        }
    }
}
```

#### 4. 运行项目

点击 Android Studio 的 Run 按钮。

### 发布到 Google Play

#### 1. 构建发布版本

```bash
./gradlew assembleRelease
```

#### 2. 签名 APK

```bash
jarsigner -verbose -sigalg SHA1withRSA -digestalg SHA1 \
  -keystore keystore.jks \
  -storepass your-password \
  -keypass your-key-password \
  app-release-unsigned.apk \
  your-key-alias
```

#### 3. 对齐 APK

```bash
zipalign -v 4 app-release-signed.apk app-release-aligned.apk
```

#### 4. 上传到 Google Play Console

1. 创建新应用
2. 填写应用信息
3. 上传 APK
4. 配置价格和分发
5. 提交审核

## Flutter 跨平台部署

### 环境要求

- Flutter SDK
- Dart SDK
- 对应平台的开发工具

### 开发步骤

#### 1. 安装 Flutter

```bash
git clone https://github.com/flutter/flutter.git -b stable
export PATH="$PATH:`pwd`/flutter/bin:$PATH"
```

#### 2. 检查环境

```bash
flutter doctor
```

#### 3. 运行应用

```bash
cd flutter_app
flutter run
```

### 构建发布版本

#### iOS

```bash
flutter build ios --release
```

#### Android

```bash
flutter build apk --release
```

#### Web

```bash
flutter build web --release
```

## 故障排查

### iOS 问题

#### 构建失败

```bash
xcodebuild clean
```

#### 签名问题

检查：
1. 证书是否有效
2. Bundle Identifier 是否正确
3. Provisioning Profile 是否匹配

### Android 问题

#### 构建失败

```bash
./gradlew clean
```

#### 签名问题

检查：
1. Keystore 是否存在
2. 密码是否正确
3. Key Alias 是否匹配
