## 模型计费规则

我们为不同层级的模型设计了差异化的计费逻辑，以平衡用户体验与系统成本。

### 🟢 Free 模型 (入门款)

> **定位**：日常闲聊、简单查询、长文阅读。主打“无感消耗”。

- **基础规则**：**固定扣除 2 分 / 次**。
- **不限长度**：不限输入、输出 Token 长度。
- **不限上下文**：不限对话历史轮数。
- **熔断机制**：
  - **阈值**：单月累计消耗 Token 数超过 **200,000**。
  - **触发后**：超出部分转为按量计费，每 1000 Tokens 额外扣除 1 分。

### 🔵 Pro 模型 (主力款)

> **定位**：复杂任务、代码生成、多轮深度对话。

Pro 模型采用 **“基础费 + 阶梯计费”** 模式：

- **单次基础费**：0.1 分 / 次。
- **阶梯系数 ($k$)**：根据单次请求总 Token 数 (含历史) 动态调整。

| Token 范围 | 系数 ($k$) | 说明 |
| :--- | :--- | :--- |
| 0 - 1k | 0.8 | 入门惠民 |
| 1k - 2k | 1.0 | 标准计费 |
| 2k - 3k | 1.2 | 中等深度 |
| 3k - 4k | 1.4 | 高阶创作 |
| 4k - 10k | 1.6 | 溢价区 |
| > 10k | 2.5 | 惩罚区 |

- **公式**：$\text{扣分} = (0.1 + (\text{Tokens}/1000) \times k) \times \text{思考倍率}$

### 🟣 Max 模型 (高端款)

> **定位**：逻辑推理、数学解题、专业分析。

Max 模型针对高成本模型采用更严格的计费：

- **单次基础费**：0.2 分 / 次。
- **阶梯系数 ($k$)**：

| Token 范围 | 系数 ($k$) |
| :--- | :--- |
| 0 - 1k | 1.2 |
| 1k - 2k | 1.4 |
| 2k - 3k | 1.6 |
| 3k - 4k | 1.8 |
| 4k - 10k | 2.5 |
| > 10k | 5.0 |

- **公式**：$\text{扣分} = (0.2 + (\text{Tokens}/1000) \times k) \times \text{思考倍率}$

## 思考模式 (Thinking Mode)

启用“思考模式”后，AI 将进行深度链式思考（Chain of Thought）。
由于该模式消耗大量服务器计算资源，该次请求产生的所有计费（基础费及按量计费部分）均将 **双倍扣除 ($\times 2.0$)**。

## 积分重置规则

- 每月 1 号 0:00，系统将重置订阅用户的积分至 3500 分。
- 上月剩余积分将清零，不累积至新月份。
