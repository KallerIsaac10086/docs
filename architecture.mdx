## 整体架构

LawChat 采用前后端分离的架构设计，支持多端部署。

```
┌─────────────────────────────────────────────────────────┐
│                        客户端层                          │
├──────────────────┬──────────────────┬──────────────────┤
│   Web 前端       │   iOS 应用       │   Android 应用    │
│   (SvelteKit)    │   (SwiftUI)      │   (Kotlin)        │
└────────┬─────────┴────────┬─────────┴────────┬─────────┘
         │                  │                  │
         └──────────────────┼──────────────────┘
                            │
┌───────────────────────────┼───────────────────────────┐
│                       网络层                          │
│                  HTTP / WebSocket                      │
└───────────────────────────┼───────────────────────────┘
                            │
┌───────────────────────────┴───────────────────────────┐
│                      后端层                            │
│                    FastAPI                            │
├───────────────────────────────────────────────────────┤
│  API 路由层                                            │
│  ├── auth (认证)         │  ├── chat (对话)          │
│  ├── membership (会员)   │  ├── models (模型)        │
│  ├── files (文件)         │  ├── cloud (云端)        │
│  ├── tools (工具)         │  ├── manage (管理)       │
├───────────────────────────────────────────────────────┤
│  业务逻辑层                                             │
│  ├── 服务层 (services)     │  ├── 工具层 (tools)    │
│  ├── 定时任务 (scheduler)  │  ├── 缓存 (cache)       │
├───────────────────────────────────────────────────────┤
│  数据访问层                                             │
│  ├── SQLAlchemy ORM       │  ├── SQLite / PostgreSQL │
└───────────────────────────────────────────────────────┘
```

## 技术栈详解

### 后端技术栈

| 技术 | 用途 | 说明 |
|------|------|------|
| FastAPI | Web 框架 | 高性能异步 Web 框架 |
| SQLAlchemy | ORM | 数据库对象关系映射 |
| SQLite/PostgreSQL | 数据库 | 默认 SQLite，支持 PostgreSQL |
| PyJWT | 认证 | JWT Token 生成和验证 |
| bcrypt | 加密 | 密码哈希 |
| httpx | HTTP 客户端 | 异步 HTTP 请求 |
| python-multipart | 文件上传 | 处理 multipart/form-data |

### 前端技术栈

| 技术 | 用途 | 说明 |
|------|------|------|
| SvelteKit | 前端框架 | 全栈前端框架 |
| Svelte 5 | UI 库 | 响应式组件库 |
| TailwindCSS | CSS 框架 | 原子化 CSS |
| Dexie.js | IndexedDB | 浏览器本地数据库 |
| TypeScript | 类型系统 | 类型安全 |
| Bits UI | UI 组件库 | 无头组件库 |
| Lucide | 图标库 | 矢量图标 |

### 移动端技术栈

| 平台 | 技术 | 说明 |
|------|------|------|
| iOS | SwiftUI | 原生 iOS 应用 |
| Android | Kotlin | 原生 Android 应用 |
| 跨平台 | Flutter | 可选的跨平台方案 |

## 核心模块说明

### 1. 认证模块 (auth)
- 用户注册、登录、登出
- JWT Token 管理
- 权限验证

### 2. 对话模块 (chat)
- 对话消息处理
- 模型调用代理
- SSE 流式响应
- 私有对话

### 3. 会员模块 (membership)
- 套餐管理
- 订阅管理
- 支付交易
- 代金券

### 4. 模型模块 (models)
- 模型配置管理
- 模型信息查询
- 热更新支持

### 5. 工具模块 (tools)
- 插件注册
- 工具执行
- 工具管理

### 6. 云端同步 (cloud)
- 对话上传/下载
- 同步状态检查
- 存储用量统计

## 数据流

### 对话流程
```
用户输入 → 前端 → 后端 API → 认证验证 → 用量检查 → 模型代理 → AI 模型
                                                  ↓
用户接收 ← 前端 ← SSE 流式响应 ← 后端 ← 响应处理
```

### 会员扣费流程
```
用户请求 → 用量检查 → 余额/套餐验证 → 扣费记录 → 允许访问
                                              ↓
                                         用量统计更新
```

## 定时任务

系统内置三个定时任务：

1. **每日维护任务** (北京时间 00:00)
   - 重置每日用量统计
   - 处理过期会员
   - 处理过期资源包

2. **每小时订阅扣费**
   - 处理自动续费扣款

3. **待处理交易清理** (每60秒)
   - 清理过期的 pending 交易
